# EmailHub 快速参考卡片

## 🎯 5 分钟快速配置

### 第一步：添加邮箱
```
仪表盘 → 邮箱管理 → 添加邮箱
├─ 邮箱地址: your-email@gmail.com
├─ 密码: [应用专用密码]
└─ 提供商: Gmail
```

**获取 Gmail 应用密码：**
https://myaccount.google.com/apppasswords

### 第二步：创建推送渠道
```
仪表盘 → 推送渠道 → 新建渠道
├─ 类型: 飞书
├─ 名称: 工作群
└─ Webhook: [从飞书群机器人获取]
```

**获取飞书 Webhook：**
飞书群 → 设置 → 群机器人 → 添加机器人 → 自定义机器人

### 第三步：创建过滤规则
```
仪表盘 → 过滤规则 → 新建规则
├─ 名称: 推送所有邮件
├─ 条件: [留空 = 匹配所有]
└─ 操作: ✓ 推送到渠道 → 选择"工作群"
```

### 第四步：启动监听
```
仪表盘 → 邮箱管理 → 点击"启动监听"
状态变为 🟢 监听中
```

### 第五步：测试
发送测试邮件 → 等待 5-10 秒 → 检查飞书群 ✅

## 🔑 获取密码/授权码

### Gmail
1. 访问 https://myaccount.google.com/apppasswords
2. 选择"邮件"和"其他设备"
3. 生成并复制密码

### Outlook
1. 访问 https://account.live.com/proofs/AppPassword
2. 创建应用密码
3. 复制密码

### QQ 邮箱
1. 邮箱设置 → 账户
2. POP3/IMAP/SMTP 服务
3. 生成授权码

### 163 邮箱
1. 邮箱设置 → POP3/SMTP/IMAP
2. 开启 IMAP 服务
3. 获取授权码

## 🔧 故障排查

### 邮箱连接失败
- ✓ 确认使用应用专用密码（不是登录密码）
- ✓ 检查 IMAP 服务是否开启
- ✓ 核对 IMAP 主机和端口

### 收不到推送
- ✓ 测试推送渠道（点击"测试"按钮）
- ✓ 检查过滤规则条件
- ✓ 确认邮箱状态为"监听中"
- ✓ 验证 Webhook URL 是否有效

### 推送延迟
- ✓ 重启邮件监听
- ✓ 检查网络连接
- ✓ 查看系统资源使用情况

### 重复推送
- ✓ 检查是否有重复规则
- ✓ 停止所有监听后重新启动
- ✓ 设置推送频率限制

## 💡 最佳实践

### ✅ 推荐做法
- 使用应用专用密码，不要用登录密码
- 设置合理的推送频率限制
- 定期检查过滤规则效果
- 使用免打扰时段避免夜间打扰
- 重要邮件单独设置高优先级规则

### ❌ 避免做法
- 不要同时监听过多邮箱（建议 ≤ 5 个）
- 不要创建过于复杂的规则
- 不要在规则中使用过于宽泛的条件
- 不要忘记定期清理旧邮件
- 不要在公共场所使用
